## 1. 产品概述
一个极简的种草社区热榜聚合页，用户打开即可看到**小红书、可颂、得物**三个平台当前最热门的话题/内容标题。  
目标：让用户“一屏扫完三大种草社区正在流行什么”，无需安装任何 App。

## 2. 目标用户
- 想快速了解潮流的上班族、学生、年轻人。用户年龄段在18~40岁之间。
- 内容创作者寻找选题灵感
- 对 Vibe Coding 感兴趣的学习者

## 3. MVP 核心功能（必须完成）
### 3.1 首页卡片网格
- 三张卡片，分别对应：**小红书、可颂、得物**
- 桌面浏览器显示为 **3 列**，手机浏览器自动变为 **1 列**（响应式）
- 每张卡片包含：
  - 平台名称 + 榜单名称（如“小红书 · 种草趋势榜”）
  - Top 10 列表，每条显示：
    - 排名（1、2、3…）
    - 标题（可点击）
    - 热度值（如“12.3w热度”）、点赞、互动量
  - 底部显示“更新于 x 分钟前”
  - 若该平台数据加载失败，显示“暂时无法获取数据”及重试按钮，不影响其他平台展示。

### 3.2 数据获取（后端）
- 后端提供 **统一接口**：`GET /api/trend/all`
- 返回三个平台的数据，格式统一（见 TECH_DESIGN）
- 数据来源：**Uapis 聚合 API**（优先），若 Uapis 不可用则降级为 **Mock 数据**
- 每个平台独立缓存，缓存时间 **600 秒（10 分钟）**，避免频繁调用 Uapis（免费版有每日次数限制）

### 3.3 交互要求
- 点击任意标题 → 在新标签页打开该内容的**原始链接**（Uapis 返回的 url 字段）
- 单平台加载失败 → 只影响该卡片，其他卡片正常显示
- 提供全页面手动刷新按钮（重新请求 `/api/trend/all`，会先读缓存，除非强制跳过）

### 3.4 部署要求
- 前端部署到 **Vercel**，后端部署到 **Railway**
- 提供公网 HTTPS 链接
- 代码托管在 **GitHub** 公开仓库

## 4. 后续版本暂不实现
- 用户登录 / 收藏 / 评论
- 历史趋势曲线
- 多级分类（如“美妆”“数码”）
- 关键词搜索

## 5. 界面设计参考
- 风格：干净、白色背景、轻微阴影、圆角卡片
- 排名 1～3 名用特殊颜色（金色、银色、铜色，或渐变背景）
- 标题 hover 时变色 + 下划线 + 字体由排名最大，并依次1~3逐渐变小
- 页脚必须包含：
  - “本站为个人学习项目，数据仅为演示作用，数据来源于第三方公开接口，非官方”
  - “数据更新频率约 10 分钟”
  - 数据来源说明：Uapis / 天行数据（按实际填写）

## 6. 非功能性需求
- 首页首次加载时间（含后端请求）< 2 秒（依赖缓存）
- 后端需记录调用 Uapis 的日志（方便排查限额问题）
- 环境变量：`UAPIS_KEY`（必填）、`CACHE_TTL`（可选，默认600）
